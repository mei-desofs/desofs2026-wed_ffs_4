# Dangerous Functionality Inventory

This document highlights application areas where legitimate functionality is security-sensitive and could cause high impact if abused, misconfigured, or implemented incorrectly.

| Functionality | Implementation location | Why it is dangerous | Required controls |
| --- | --- | --- | --- |
| Authentication and token issuing | `src/main/java/com/desofs/auth/controller/AuthController.java`, `src/main/java/com/desofs/auth/service/AuthService.java`, `src/main/java/com/desofs/security/JwtUtil.java` | Issues JWT and refresh tokens that grant access to protected API operations. Weak validation, weak secrets, token leakage, or brute-force attacks could lead to account compromise. | Password hashing with bcrypt, generic failed-login errors, failed-attempt lockout, JWT signature and expiration validation, refresh-token rotation, audit logging of authentication events. |
| Logout and session termination | `src/main/java/com/desofs/auth/controller/AuthController.java`, `src/main/java/com/desofs/security/TokenBlacklistService.java`, `src/main/java/com/desofs/auth/service/RefreshTokenService.java` | If tokens remain usable after logout, a stolen token can continue to access the application. | Blacklist terminated access tokens, revoke stored refresh tokens on logout, validate blacklist status on authenticated requests. |
| File upload | `src/main/java/com/desofs/attachment/controller/AttachmentController.java`, `src/main/java/com/desofs/attachment/service/AttachmentService.java` | Accepts untrusted user-controlled files. Abuse could cause malware storage, MIME confusion, path traversal, storage exhaustion, or image pixel flood attacks. | Extension allowlist, MIME validation, maximum file size, upload rate limit, image pixel-count validation, safe filename handling, UUID stored filenames, storage path normalization. |
| File download | `src/main/java/com/desofs/attachment/controller/AttachmentController.java`, `src/main/java/com/desofs/attachment/service/AttachmentService.java` | Serves stored user-uploaded content. Incorrect authorization or path handling could expose files across projects or users. | Authenticated download endpoint, project membership checks, stored-name path normalization, no direct public file serving. |
| Role changes | `src/main/java/com/desofs/user/controller/UserController.java`, `src/main/java/com/desofs/user/service/UserService.java`, `src/main/java/com/desofs/config/SecurityConfig.java` | Role updates can grant elevated privileges such as manager or admin access. Incorrect authorization could lead to privilege escalation. | Admin-only role update route, Spring Security path authorization, audit logging for role changes. |
| Project membership management | `src/main/java/com/desofs/project/controller/ProjectMemberController.java`, `src/main/java/com/desofs/project/service/ProjectMemberService.java` | Adding or removing project members changes access to project tasks, comments, and attachments. Abuse could expose confidential project data. | Role-based access checks, project membership validation, repository checks scoped to project ID. |
| Task, comment, and project authorization | `src/main/java/com/desofs/task/service/TaskService.java`, `src/main/java/com/desofs/comment/service/CommentService.java`, `src/main/java/com/desofs/project/service/ProjectService.java` | These operations expose and modify team data. Missing ownership or membership checks could allow horizontal privilege escalation. | Project membership checks, role checks for manager/admin operations, author checks for comment changes, soft-delete handling. |
| Audit logging | `src/main/java/com/desofs/audit/service/AuditService.java`, `src/main/java/com/desofs/config/GlobalExceptionHandler.java` | Logs support incident response but can leak sensitive data or be abused for log injection if unsafe content is recorded. | Do not log passwords or tokens, record security-relevant events, keep client error responses generic for unexpected errors. |
| Cryptographic and TLS configuration | `src/main/resources/application.yml`, `src/main/resources/application-prod.yml`, `src/main/java/com/desofs/security/JwtUtil.java` | JWT secrets and TLS keystores protect authentication and transport security. Weak, leaked, or expired keys can compromise confidentiality and integrity. | Store production secrets outside source control, use strong JWT secret material, configure production TLS keystore through environment variables, document approved algorithms and cipher suites. |
